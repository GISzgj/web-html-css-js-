#### 标签分类

按照标签是否独占一行来划分

##### 行级标签

```
特点
	上一行有空间,就在上一行显示,如果没有空间,才会另起一行(font,em)
纯粹的行级标签
	span标签:默认不可以换行,主要用于存放少量的数据
```

##### 块级标签

```
特点
	不管上一行有没有空间,直接另起一行显示(p,h3)
纯粹的块级标签
	div标签:默认是自动换行,主要用于布局(以后的主流布局方式就是div+css)
```



#### 图片媒体标签

##### 图片标签

```
标签名 img
	重要的属性 
		src 	图片的路径,推荐使用相对路径
				如果图片和当前文件在同一个目录下,使用 图片名(./图片名)
				如果图片在当前文件的下一级目录,使用 目录名/图片名(./目录名/图片名)
				如果图片在当前文件的上一级目录,使用 ../图片名
				
	其他属性
		width:宽度
		height:高度
		alt:在图片无法显示的时候,替代图片
		title:鼠标经过图片的提示文字
```

##### 音频标签

```
 标签名 audio
 	src	设置或返回音频/视频元素的当前来源。 推荐使用相对路径
	controls	设置或返回音频/视频是否显示控件（比如播放/暂停等）
```

##### 视频标签

```
 标签名 video
 	src	设置或返回音频/视频元素的当前来源。 推荐使用相对路径
	controls	设置或返回音频/视频是否显示控件（比如播放/暂停等）
	height 视频的高度
```

#### 布局相关标签

```
概念
	HTML5 提供了新的语义元素来明确一个Web页面的不同部分
	之前: <div id="nav"> 导航栏 <div class="header">头部 <div id="footer"> 尾部
	现在: <nav> 导航栏  <header>头部  <footer>尾部
	
	这些标签也叫语义化标签,没有任何样式,主要用于划分不同的区域,便于页面布局
	
组成
	<header>元素描述了文档的头部区域
	<footer> 元素描述了文档的底部区域.
	<main> 元素描述了文档的主体区域.
	<aside> 标签定义页面主区域内容之外的内容（比如侧边栏）。
	<nav> 标签定义导航链接的部分
	<section> 标签定义文档中的节（section、区段）  用的比较少
	<article> 标签定义独立的内容。.               用的比较少

注意
	这些标签等同于div,是块级元素,用于划分区域的,本身没有任何样式,主要跟css结合起来布局页面
```



#### 表单相关标签

```
form 表单标签 定义表单要提交的范围和提交的地址
	重要属性
		action:提交的地址,默认提交到当前页面
		method:规定用于发送表单数据的 HTTP 方法。(表单的提交方式)
			get:把数据拼接到地址栏后面(以名称和值的成对拼接),相对的不安全,只能提交少量数据(默认就是get方式)
			post:把数据拼接在请求体中,相对的安全,可以提交大量数据,主要用于文件的上传或者提交一些敏感数据

input 输入项标签 根据type属性值的不同,显示不同的样式,让用户输入或者选择
	重要的属性
		type
			text        文本框(默认)
			password    密码框(里面的内容不以明文显示)
			radio       单选框
			checkbox    复选框
			file        文件框(用于上传文件)
			submit      提交按钮(提交页面数据)
			reset       重置按钮(恢复页面刚打开时的效果)
			button      普通按钮(结合js一起使用)

		checked
			checked     单选和复选框的默认选中状态

select 下拉选择框  对于取值是固定的几个,可以考虑使用下拉选择框
	option 选择框的选项   他不是属性,是select的子标签
		重要属性
			selected
				selected    设置下拉选择框的默认选中


通用属性
	name
		表单项元素要想把数据提交到后台,必须要有name属性
	value
		对于单选和复选框,需要指定value属性的值,用于在后台区分用户选择了哪一个选项
		可以设置下拉选择框选择项option标签的值,设置之后,提交到后台使用的就是value里面的值,否则使用的标签体的值
```
